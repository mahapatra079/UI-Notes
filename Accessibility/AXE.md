# AXE (web accessibility)

Accessibility is done to enable for the blind people to access the application.

## Tools

- **JAWS** - Screen Reader that helps people to read content on the screen
- **Narrator Tool** - Built-in Windows screen reader for accessibility testing

## Tools For Checking

- AXE Dev Tools => This is a browser extension that helps to identify accessibility issues on the web page.
- AXE CLI => This is a command line tool that helps to identify accessibility issues on the web page.
- AXE API => This is a REST API that helps to identify accessibility issues on the web page.
- AXE Core => This is a JavaScript library that helps to identify accessibility issues on the web page.
- AXE Pro => This is a paid version of AXE that provides additional features and support.

## WCAG 2.0
WCAG 2.0 is a set of guidelines for making web content more accessible to people with disabilities.

## Conclusion

Accessibility is important to ensure that everyone can access and use the web, regardless of their abilities. By using tools like AXE, we can identify and fix accessibility issues to create a more inclusive web experience for all users.
